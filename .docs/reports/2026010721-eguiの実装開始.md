## 目的

- 既存の CLI に加えて、最小の GUI 入力でコマンド起動できる土台を用意する

## 実装内容

- UI 実装のモジュール構成を導入

  - src/ui.rs を削除し、src/ui/\* に分割
  - src/ui/launcher.rs
    - コマンド名の入力と Enter による実行
    - 設定からコマンド定義を取得し、runner を呼び出してプロセスを起動
  - src/ui/initialize.rs
    - egui のフォント定義を差し替え

- エントリーポイントの動作を更新

  - 引数なし起動時に UI を起動する
  - 簡易 CLI の list / run-first / run は従来どおり利用できる

- 設定読み込み API を UI 向けに更新

  - config::load_settings() の戻り値を Commands から Settings に変更
  - Settings は Commands を保持し、commands() で取り出せる

- ログレベルを調整
  - Debug ビルドは Info
  - Release ビルドは Warn

## 仕様書への反映

- .docs/specs/project.md
  - UI の配置と起動方法（引数なしで UI 起動）を更新
  - ログレベルの仕様を更新
- .docs/specs/history.md
  - 上記の仕様変更履歴を追記

## 影響範囲と注意点

- UI は eframe/egui を用いるため、実行環境によっては描画やフォントの挙動が変わる
- src/ui/initialize.rs で Windows フォントへのパスを include_bytes! しているため、実行環境とフォント配置に依存する
